# Предобработка изображений средствами OpenCV

Программа на Python исследует основные методы подготовки изображений для задач компьютерного зрения: преобразование цветовых пространств, сглаживание, повышение резкости, морфологические операции и удаление шума.

## Возможности программы

- загрузка трёх тестовых изображений: портрет, городской пейзаж и изображение с текстом;
- корректное отображение изображений, считанных OpenCV в формате BGR;
- сравнение RGB, Grayscale и каналов HSV;
- применение GaussianBlur, MedianBlur и AverageBlur с ядрами 5x5, 11x11 и 21x21;
- повышение резкости с помощью ядра свёртки;
- бинаризация и выполнение erosion, dilation, opening и closing;
- добавление гауссова шума и шума «соль-перец»;
- сравнение фильтров на зашумлённых изображениях;
- сохранение итоговых таблиц-сеток в папку `outputs`.

## Структура программы

Основная логика финальной версии разделена на классы:

- `ImageItem` — хранение имени, пути и массива изображения;
- `ImageRepository` — загрузка тестовых изображений;
- `ColorSpaceProcessor` — преобразования RGB, Grayscale и HSV;
- `FilterProcessor` — размытие и повышение резкости;
- `NoiseProcessor` — добавление двух типов шума;
- `MorphologyProcessor` — бинаризация и морфологические операции;
- `Visualizer` — вывод и сохранение таблиц-сеток;
- `PreprocessingExperiment` — общий сценарий проведения эксперимента.

Основной файл программы:

```text
main.py
```

## Требования

Для запуска потребуется:

- Python 3.10 или новее;
- OpenCV;
- NumPy;
- Matplotlib.

## Установка библиотек

В терминале в папке проекта выполнить:

```bash
pip install opencv-python numpy matplotlib
```

## Подготовка изображений

В корне проекта необходимо создать папку `images` и поместить в неё три изображения со следующими именами:

```text
project/
├── main.py
├── images/
│   ├── portrait.jpg
│   ├── city.jpg
│   └── text.jpg
└── outputs/
```

Назначение файлов:

- `portrait.jpg` — портрет человека;
- `city.jpg` — городской пейзаж;
- `text.jpg` — фотография текста, страницы или вывески.

Изображения можно заменить своими, сохранив имена файлов либо изменив пути в функции `main()`.

## Запуск программы

В терминале из корневой папки проекта выполнить:

```bash
python main.py
```

После запуска программа последовательно отображает результаты обработки и сохраняет их в каталог `outputs`.

## Результаты обработки

После выполнения программы в папке `outputs` формируются файлы:

```text
01_originals.png
02_color_spaces_portrait.png
02_color_spaces_city.png
02_color_spaces_text.png
03_smoothing_filters.png
04_sharpening.png
05_morphology.png
06_gaussian_noise_reduction.png
06_salt_pepper_noise_reduction.png
07_final_comparison.png
```

Назначение файлов:

- `01_originals.png` — три исходных изображения;
- `02_color_spaces_*.png` — RGB, Grayscale и отдельные каналы HSV;
- `03_smoothing_filters.png` — сравнение трёх сглаживающих фильтров и трёх размеров ядра;
- `04_sharpening.png` — изображение до и после повышения резкости;
- `05_morphology.png` — результаты морфологических операций;
- `06_gaussian_noise_reduction.png` — фильтрация гауссова шума;
- `06_salt_pepper_noise_reduction.png` — фильтрация шума «соль-перец»;
- `07_final_comparison.png` — итоговая сравнительная таблица для отчёта.

## Как проверить качество работы

Для оценки результата необходимо проверить:

1. Корректно ли отображаются цвета исходных изображений после преобразования BGR в RGB.
2. Видны ли различия между RGB, Grayscale и каналами HSV.
3. Усиливается ли размытие при увеличении ядра с 5x5 до 21x21.
4. Повышается ли визуальная чёткость границ после применения фильтра резкости.
5. Удаляет ли opening мелкие белые шумы, а closing — мелкие чёрные разрывы в бинарной маске.
6. Лучше ли GaussianBlur очищает гауссов шум, а MedianBlur — шум «соль-перец».
7. Сформирован ли файл `07_final_comparison.png`, содержащий итоговое сравнение методов.

Для количественной оценки удаления искусственно добавленного шума можно сравнить отфильтрованное изображение с исходным изображением по показателям MSE и PSNR. Чем меньше MSE и выше PSNR, тем ближе результат фильтрации к исходному изображению.

## Возможные проблемы

### Изображение не найдено

Следует проверить, что папка `images` находится рядом с файлом `main.py`, а имена файлов совпадают с указанными в программе.

### Цвета на изображении выглядят неправильно

При самостоятельном изменении визуализации необходимо учитывать, что OpenCV загружает цветное изображение в порядке BGR, а Matplotlib отображает RGB.

### Окно с графиками не появляется

При запуске в Jupyter Notebook изображения выводятся внутри блокнота. При запуске обычным Python-скриптом должны открываться окна Matplotlib; при необходимости можно оставить только сохранение PNG в папку `outputs`.
